
# actions (активности)

Таблица хранит в себе информацию о **действиях с заказами** (*создание*, *отмена* или *доставка*)

**Колонки таблицы**:
* *`order_id`* - ID заказа
* *`user_id`* - ID пользователя
* *`action`* - действие, выполненное с заказом. Может принимать следующие значения:
	* *`create_order`* - создание заказа
	* *`cancel_order`* - отмена заказа
	* *`deliver_order`* - заказ доставлен
* *`time`* - время активности
<br>

# orders (заказы)

Таблица хранит в себе информацию о **заказах пользователей** (какие товары заказал пользователь)

**Колонки таблицы**:
* *`order_id`* - ID заказа
* *`order_time`* - время заказа (совпадает с действием *`create_order`* таблицы *`actions`* в его колонке *`action`* )
* *`product_ids`* - список из ID товаров, добавленных в заказ
<br>

# products (товары)

Таблица хранит в себе информацию о **названии товара** и его **стоимости**

**Колонки таблицы**:
* *`product_id`* - ID товара
* *`name`* - название товара
* *`price`* - стоимость товара (в рублях)
<br>

# events (события)

Таблица хранит в себе информацию о **действиях пользователей** (*регистрация*, *просмотр товара*, *добавление в корзину*, *покупка*)

**Колонки таблицы**:
* *`event_id`* - ID события
* *`user_id`* - ID пользователя
* *`event_type`* - действие, делающее пользователь. Может принимать следующие значения:
	* *`registration`* - регистрация пользователя
	* *`view_product`* - просмотр товара
	* *`add_to_cart`* - добавление в корзину
	* *`purchase`* - покупка
* *`event_time`* - время события
* *`device`* - устройство, с которого было выполнено событие. Может принимать следующие значения:
	* *`mobile`* - мобильное устройство
	* *`desktop`* - персональный компьютер / ноутбук
<br>

# users (пользователи)

Таблица хранит в себе информацию о **личных данных пользователя**

**Колонки таблицы**:
* *`user_id`* - ID пользователя
* *`birth_date`* - день рождения
* *`gender`* - пол пользователя. Может принимать следующие значения:
	* *`male`* - мужчина
	* *`female`* - женщина
* *`city`* - город проживания. Может принимать следующие значения:
	* *`Moscow`* - Москва
	* *`Saint Petersburg`* - Санкт-Петербург
	* *`Novosibirsk`* - Новосибирск
	* *`Kazan`* - Казань
	* *`Yekaterinburg`* - Екатеринбург
* *`traffic_source`* - откуда пользователь попал и зарегистрировался (из какого источника). Может принимать следующие значения:
	* *`organic`* - самостоятельно нашёл и зарегистрировался
	* *`ads`* - через рекламную ссылку
	* *`referral`* - через реферальную ссылку от другого пользователя
